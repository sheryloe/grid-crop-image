## 들어가며

이미지 한 장을 여러 조각으로 나눠 저장해야 하는 순간은 생각보다 자주 찾아옵니다. 블로그에 올릴 스크린샷을 구간별로 잘라야 할 때도 있고, 긴 이미지를 문서에 맞게 정리해야 할 때도 있습니다. 웹툰 컷처럼 세로로 긴 이미지를 여러 장으로 나눠 저장해야 하는 경우도 있고요.

문제는 이런 작업이 자주 반복되는데도, 막상 딱 맞는 도구를 찾기는 쉽지 않다는 점이었습니다. 기능은 많지만 너무 무겁거나, 반대로 너무 단순해서 반복 작업에 쓰기 어려운 경우가 많았습니다. 그래서 이번에는 필요한 기능만 정확하게 담은 가벼운 Windows 이미지 분할 툴을 직접 만들어보기로 했습니다.

이번 글은 `Grid Crop Image`를 왜 만들게 됐는지, 어떤 문제를 해결하고 싶었는지, 그리고 첫 구현 범위를 어떻게 정했는지를 정리한 1단계 기록입니다.

## 왜 새로 만들 생각을 했을까

처음에는 기존 이미지 편집 툴로도 어느 정도 해결이 가능하다고 생각했습니다. 하지만 실제로 반복 작업을 해보면 불편한 지점이 계속 쌓였습니다.

- 캡처 이미지를 바로 잘라 저장하고 싶은데 매번 무거운 편집 프로그램을 켜야 했습니다.
- 한 이미지 안에서 여러 영역을 나눠 저장하려면 같은 동작을 반복해서 해야 했습니다.
- 클립보드에 있는 이미지를 바로 붙여넣어 쓰고 싶은데, 먼저 파일로 저장해야 하는 경우가 많았습니다.
- 결과 파일명이 일정하지 않아 작업 후 정리하는 시간이 따로 들었습니다.
- 나중에 같은 유형의 이미지를 다시 자르려면 좌표를 처음부터 다시 맞춰야 했습니다.

결국 필요한 것은 화려한 편집 기능이 아니라, 빠르게 열고 바로 나누고 곧바로 저장할 수 있는 흐름이었습니다.

## 기존 도구들이 아쉬웠던 이유

이미지 편집 툴 자체는 많습니다. 다만 이번 작업처럼 "한 장의 이미지를 여러 조각으로 반복해서 저장하는 작업"에 맞춰 보면 아쉬운 부분이 분명했습니다.

### 1. 기능은 많지만 너무 무겁다

포토샵이나 대형 편집 툴은 분명 강력합니다. 하지만 단순히 여러 영역을 잘라 저장하려는 상황에서는 실행 속도와 조작 단계 자체가 부담으로 느껴졌습니다. 짧고 반복적인 작업일수록 이 차이가 더 크게 체감됐습니다.

### 2. 클립보드 중심 작업 흐름이 약하다

실제로는 캡처 후 바로 붙여넣어 작업을 이어가고 싶은 경우가 많습니다. 그런데 많은 툴은 파일 저장 이후 작업을 전제로 움직입니다. 별것 아닌 것 같아도, 이 한 단계가 반복되면 생산성이 꽤 떨어집니다.

### 3. 다중 영역 저장이 생각보다 번거롭다

이번 프로젝트에서 가장 중요한 기능은 한 번에 여러 사각형을 지정하고, 그 순서대로 저장하는 것입니다. 그런데 일반 편집 툴에서는 이 흐름이 직관적이지 않거나 수동 작업이 많았습니다.

### 4. 설정을 재사용할 수 있어야 했다

비슷한 레이아웃을 반복해서 자르는 작업에서는 "한 번 정한 영역을 다시 쓸 수 있는가"가 중요합니다. 단순히 크롭이 되는 것보다, 다음 작업 때도 같은 좌표를 불러올 수 있는지가 실용성 차이를 만들었습니다.

## 그래서 어떤 툴을 만들기로 했나

목표는 처음부터 분명했습니다. 복잡한 편집기가 아니라, Windows에서 가볍게 실행되는 이미지 분할 전용 유틸리티를 만드는 것이었습니다.

핵심 목표는 아래처럼 정리했습니다.

- 로컬 이미지 파일을 바로 열 수 있을 것
- `Ctrl + V`로 클립보드 이미지를 바로 붙여넣을 수 있을 것
- 여러 개의 사각형 영역을 직접 그리고 이동할 수 있을 것
- 확정한 순서대로 개별 이미지 파일을 저장할 수 있을 것
- 같은 작업을 반복할 수 있도록 설정 저장과 불러오기가 가능할 것
- 최종적으로는 EXE로 빌드해 Python 설치 없이도 실행할 수 있을 것

한마디로 정리하면, "반복 분할 작업에 최적화된 가벼운 도구"를 만들고 싶었습니다.

## 기술 스택은 왜 Python + Tkinter + Pillow였을까

이번 프로젝트는 거창한 구조보다 빠른 구현과 빠른 수정이 더 중요했습니다. 그런 기준으로 보면 Python이 가장 잘 맞았습니다.

### Python을 선택한 이유

이미지 처리 로직을 붙이기 좋고, 작은 유틸리티를 빠르게 형태로 만드는 데 유리했습니다. 반복해서 기능을 추가하고 수정해야 하는 프로젝트에서는 개발 속도 자체가 큰 장점이 됩니다.

### Tkinter를 사용한 이유

Windows에서 간단한 GUI를 빠르게 만들고 싶었고, 외부 프레임워크 의존성을 과하게 늘리고 싶지 않았습니다. Tkinter는 버튼, 캔버스, 스크롤, 키 이벤트 같은 기본 구조를 바로 구성하기에 충분히 실용적이었습니다.

### Pillow를 사용한 이유

이미지 열기, 확대/축소, 크롭, 저장, 클립보드 처리까지 한 흐름으로 다루기 좋았습니다. 지원 포맷도 넓고, 이번 프로젝트 규모에서는 가장 안정적인 선택이었습니다.
