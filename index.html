스크린샷 자르기와 이미지 분할을 더 빠르게.

Grid Crop Image는 Windows에서 한 장의 이미지를 여러 개의 직사각형 영역으로 나눠 저장할 수 있는 가벼운 데스크톱 앱입니다. 로컬 파일 열기, Ctrl + V 클립보드 이미지 붙여넣기, 다중 사각형 크롭, JSON 설정 저장, 순차 파일 저장까지 한 흐름으로 처리할 수 있습니다.

저장소 보기 사용 가이드 읽기
Ctrl + V 클립보드 이미지 붙여넣기와 파일 목록 감지를 지원합니다.
JSON Save 크롭 좌표와 줌 상태를 저장하고 다시 불러올 수 있습니다.
Sequential Export 지정한 사각형 순서대로 이미지 파일을 자동 저장합니다.
Why This Tool

왜 Grid Crop Image가 필요한가

포토샵 같은 무거운 툴 없이 빠르게 스크린샷을 나누고 싶을 때, 긴 이미지를 구간별로 저장해야 할 때, 같은 크롭 레이아웃을 반복해서 써야 할 때 이 앱이 가장 효율적입니다.

Windows 이미지 크롭 프로그램

가볍게 실행되는 Tkinter 기반 앱이라 반복 작업에서 실행 부담이 적습니다.

스크린샷 자르기 도구

캡처 후 바로 붙여넣고 여러 조각으로 나눠 저장하는 흐름에 맞춰져 있습니다.

반복 크롭 자동화

JSON 설정 저장과 좌표 보정 덕분에 동일한 분할 작업을 다시 수행하기 쉽습니다.

Core Features

핵심 기능

검색 키워드 수준이 아니라 실제 사용 흐름에 맞는 기능만 정리했습니다.

클립보드 이미지 붙여넣기

`Ctrl + V`로 이미지 데이터를 바로 불러오고, 파일 복사 상황에서는 첫 번째 지원 이미지를 자동으로 감지합니다.

다중 사각형 크롭

여러 개의 크롭 사각형을 생성, 이동, 삭제, 초기화할 수 있으며 저장 순서도 일관되게 유지됩니다.

JSON 설정 저장과 불러오기

원본 이미지 경로, 출력 폴더, 줌, 좌표, 설정 상태까지 저장하고, 크기가 달라지면 비율 보정도 수행합니다.

확대, 축소, 맞춤 보기

Ctrl + 마우스휠 확대/축소, Shift + 마우스휠 가로 스크롤로 긴 이미지도 빠르게 조정할 수 있습니다.

순차 파일 저장

확정한 사각형 순서대로 `rect_01`, `rect_02` 형식으로 저장하고, 중복 파일명도 자동 회피합니다.

Windows EXE 빌드

PyInstaller로 `AutoCropSplitter.exe`를 만들 수 있어 Python 환경이 없는 곳에도 배포하기 좋습니다.

Workflow

작업 흐름은 단순하게 유지했습니다

이미지를 여는 순간부터 저장까지의 클릭 수와 이동 경로를 짧게 유지하는 데 초점을 맞췄습니다.

추천 작업 순서

로컬 파일을 열거나 클립보드 이미지를 붙여넣고, 저장 폴더를 확인한 다음, 사각형을 만든 뒤 `설정`으로 순서를 확정하고 `분할 시작`으로 저장합니다. 이 구조 덕분에 저장 조건이 명확하고, 왜 버튼이 비활성화되어 있는지도 이해하기 쉽습니다.

  1. 01
    이미지 열기 또는 Ctrl + V 붙여넣기
  2. 02
    저장 폴더 확인 및 변경
  3. 03
    드래그로 크롭 사각형 생성 및 수정
  4. 04
    설정 후 분할 시작으로 순차 저장
Tech Notes

현재 프로젝트 기준 요약

검색 방문자가 빠르게 이해할 수 있도록 앱 특성과 배포 포인트를 함께 정리했습니다.

지원 환경

Windows 10 이상, Python 3.11 이상, Pillow 기반 이미지 처리, Tkinter 기반 GUI 구조입니다.

저장소와 실행 이름

저장소 이름은 grid-crop-image, 프로그램 실행 창 제목과 EXE 이름은 Auto Crop Splitter입니다.

FAQ

자주 묻는 질문

Search Console과 GitHub Pages에서 읽기 좋은 Q&A 구조로 같이 배치했습니다.

Grid Crop Image는 어떤 작업에서 가장 효율적인가요?

스크린샷 자르기, 긴 이미지 분할, 다중 영역 크롭, 반복 레이아웃 저장 작업에 가장 효율적입니다.

클립보드 이미지도 바로 붙여넣을 수 있나요?

가능합니다. `Ctrl + V`로 이미지 데이터를 바로 불러오고, 복사된 파일 목록에서 지원 이미지를 찾는 흐름도 지원합니다.

JSON 설정 저장은 왜 필요한가요?

같은 크롭 좌표를 반복 작업에 재사용할 수 있고, 이미지 크기가 달라져도 비율 보정으로 다시 적용하기 쉬워집니다.

어디서 다운로드하거나 코드 확인을 할 수 있나요?

GitHub 저장소에서 코드와 README를 바로 확인할 수 있으며, EXE 빌드 방법도 저장소 문서에 정리돼 있습니다.
